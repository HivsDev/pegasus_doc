          • Overview System Control, based on chip characteristics, completes the reset and basic initialization of each hardware component. It is also responsible for initializing and de-initializing each business module of the MPP (Media Process Platform) system, managing the working status of each MPP system business module, providing the current MPP system version information, and providing large-block physical memory management functions. Before an application starts MPP services, it must complete MPP system initialization. Similarly, after the application exits MPP services, it must complete MPP system de-initialization to release resources. > Notice: ## Important Concepts ### Low Latency Low latency means that after the image is written to the specified number of lines to DDR, the front-end module reports an interrupt and sends the image to the back-end module for processing. This can reduce pipeline latency, and the front-end and back-end modules use a specific mechanism to ensure that the image is written before being read, so no image read errors occur. > Notice: >- Enabling low latency on each module may conflict with certain specific feature combinations. For the specific enabling method and corresponding limitations, refer to the relevant low latency descriptions in each module chapter.¶
          • Functional Description ### Video Buffer Pool The video buffer pool mainly provides large-block physical memory management functions for media services, responsible for memory allocation and recycling, fully leveraging the role of the memory buffer pool to ensure reasonable use of physical memory resources across various media processing modules. A group of cache blocks of the same size and contiguous physical addresses form a video buffer pool. The public video buffer pool must be configured before system initialization. Depending on the service, the number of public buffer pools and the size and quantity of cache blocks may vary. All video input channels can obtain video cache blocks from the public video buffer pool to store captured images. As shown in Figure 1, VI obtains cache block Bm from the public video buffer pool B. Cache block Bm is sent from VI to VPSS. After being processed by VPSS, the input cache block Bm is released back to the public video buffer pool. Assuming the VPSS channel working mode is USER, VPSS channel 0 obtains cache block Bi from public video buffer pool B as the output image cache buffer and sends it to VENC. VPSS channel 1 obtains cache block Bk from public video buffer pool B as the output image cache buffer and sends it to VO. Bi is released back to the public video buffer pool after VENC encoding is complete, and Bk is released back after VO display is complete.
