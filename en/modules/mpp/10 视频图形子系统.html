          • Overview VGS is the Video Graphics Sub-System. It supports processing a single input image, including scaling, pixel format conversion, video storage format conversion, compression/decompression, Cover, OSD, Mosaic, draw line, rotation, luminance and statistics, Corner_rect, stitching, and other operations. ## Functional Description ### Basic Concepts - job A structure for VGS to manage tasks. A job can contain multiple tasks. VGS ensures that tasks are executed in the order they are added to the job and that all tasks in a job are submitted to hardware for execution at once. Users can set the module parameter (g_max_vgs_job) when loading the VGS kernel module to configure the maximum number of jobs supported by VGS. The number of jobs ranges from [20, 400], with a default of 128. - task Performs one or more specific operations on an image, such as applying OSD, scaling, or pixel format conversion. Users can set the module parameter (g_max_vgs_task) when loading the VGS kernel module to configure the maximum number of tasks supported by VGS. The number of tasks ranges from [20, 800], with a default of 200. - node The minimum execution unit of VGS hardware. One task corresponds to one or more nodes. A node contains the information required by the hardware, such as the source image address, destination image address, operation type, etc., organized in the format required by the hardware. Users can set the module parameter (g_max_vgs_node) when loading the VGS kernel module to configure the maximum number of nodes supported by VGS. The number of nodes ranges from [20, 800], with a default of 200. - HANDLE A task handle that identifies a job. ### Functional Description The main functions of VGS include scaling, OSD, Cover, Mosaic, draw line, pixel format conversion, video storage format conversion, compression/decompression, rotation, luminance and statistics, Corner_rect, stitching, and low-power strategy. - Scaling VGS supports scaling an image, with a maximum magnification of 16x in both width and height, and a maximum reduction of 15x in both width and height. Single-component (Y) scaling is supported. - OSD VGS supports overlaying a bitmap on an image. The invert color feature of the bitmap can be set. Batch OSD operations are supported. Note: - When applying multiple OSDs (batch OSD), it is implemented internally by submitting multiple tasks. Each task uses the image with the OSD applied by the previous task as the source image for the current task's OSD processing. - Cover VGS supports covering an area of an image. The shape of the covered area can be a rectangle or an arbitrary quadrilateral. The cover color is solid. Batch Cover operations are supported. Note: - When applying multiple Covers (batch Cover), it is implemented internally by submitting multiple tasks. Each task uses the image with the Cover applied by the previous task as the source image for the current task's Cover processing. - Mosaic VGS supports applying mosaic to an image. The mosaic block sizes available are 8x8, 16x16, 32x32, and 64x64. The mosaic area shape is rectangular, and each mosaic block is a solid color. Batch Mosaic operations are supported. Note: When applying multiple Mosaics (batch Mosaic), it is implemented internally by submitting multiple tasks. Each task uses the image with the Mosaic applied by the previous task as the source image for the current task's Mosaic processing. - Corner_rect VGS supports overlaying Corner_rect operations on an image. Corner_rect types can be corner frames or solid-line frames. Batch Corner_rect operations are supported. Note: - When applying multiple Corner_rects (batch Corner_rect), it is implemented internally by submitting multiple tasks. Each task uses the image with the Corner_rect applied by the previous task as the source image for the current task's Corner_rect processing. When applying batch Corner_rect, all Corner_rects share the same color and type. - Only Hi3403V100 supports this feature. - Stitching VGS supports horizontal stitching of up to 4 images. The number of images can be configured. Note: - The compression type, width/height, pixel format, and other information of stitched input images must be consistent. Stitching of compact segment compressed images is not supported. - Only Hi3403V100 supports this feature. - Draw Line VGS supports drawing lines on an image. Batch draw line operations are supported. Note: - When drawing a line, if the X coordinates of the start and end points are the same, the Y coordinates of the start and end points remain unchanged, and the X coordinates both increase by the line width to the right. - When drawing a line, if the X coordinates of the start and end points are different, the X coordinates remain unchanged, and the Y coordinates of the start and end points both extend downward by the line width (i.e., X remains the same, Y increases downward by the line width). - Pixel Format Conversion VGS supports input and output pixel formats including semi-planar 420, semi-planar 422, and single-component (Y). It supports format conversion between semi-planar 420 and semi-planar 422, and conversion from semi-planar 420 and semi-planar 422 to single-component (Y) format. The UV order can be adjusted during pixel format conversion. - Video Storage Format Conversion VGS supports conversion from Tile format to linear format. - Compression/Decompression VGS supports compression/decompression of semi-planar 422, semi-planar 420, and single-component (Y) formats in linear format. It supports decompression of semi-planar 420 format in Tile format. - Luminance and Statistics Calculates the sum of luminance information in a specified region of an image. Batch statistics are supported. - Rotation VGS supports rotation of 0, 90, 180, and 270 degrees. - Low-Power Strategy The VGS module uses a low-power strategy. When VGS is not working, the VGS clock is turned off. At this time, manually reading or writing VGS module registers may result in read/write errors or hangs. The low-power strategy can be enabled (g_save_power_en=1) or disabled (g_save_power_en=0) by setting the module parameter (g_save_power_en). Only supports this module parameter.
