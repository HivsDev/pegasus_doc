    title: "Overview" source: /sessions/sharp-sweet-allen/mnt/hi3403-build/pegasus/docs/zh-CN/MPP Media Processing Software V5.0 Development Reference/13 Procdebuginformation(13.1--13.15).md --- # proc Debug Information ## Overview Debug information is implemented via the Linux proc file system, which reflects the current system runtime status in real time. The recorded information can be used for problem diagnosis and analysis. File Directory /proc/umap File List

    File Name Description
    sys Records the current usage of the SYS module.
    logmpp Records the current debug level of each module, for internal debugging use.
    chnl CHNL module status.
    vgs Video scaling processing unit status information.
    h265e During H.265 encoding, statistics of encoding attributes, status, and historical information for each channel.
    h264e During H.264 encoding, statistics of encoding attributes, status, and historical information for each channel.
    jpege During JPEG encoding, statistics of encoding attributes, status, and historical information for each channel.
    rgn Region management information for video overlay OSD.
    venc Video encoder information.
    vdec Video decoder information.
    vfmw_ctrl Common information for each channel during video decoding.
    vfmw_stm Stream information for each channel during video decoding.
    vfmw_syn Syntax information for each channel during video decoding.
    vfmw_dec Hardware configuration information for each channel during video decoding.
    vpss Video preprocessing module information.
    aenc Audio encoding information.
    adec Audio decoding information.
    acodec Acodec volume information.
    vda Video detection and analysis module information.
    avs Stitching processing module information

    Information Viewing Methods - On the console, the cat command can be used to view information, e.g., cat /proc/umap/venc; other common file operation commands can also be used, e.g., cp /proc/umap/ ./ -rf, to copy the proc files under umap to the current directory. - In applications, the above files can be treated as ordinary read-only files for read operations, e.g., using fopen, fread, etc. > Note: >The following 2 cases should be noted for parameter descriptions:

    • For parameters with values {0, 1}, if the specific value-meaning correspondence is not listed, then 1 indicates affirmative and 0 indicates negative.
