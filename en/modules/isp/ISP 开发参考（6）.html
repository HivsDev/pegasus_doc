The corresponding API interfaces must be called after calling the ss_mpi_isp_init interface. ## Sharpen ### Function Description The Sharpen module is used to enhance image clarity, including adjusting the sharpening attributes of image edges and enhancing the clarity of image details and textures. It can independently control the sharpening strength of directional edges and non-directional detail textures. Additionally, it can control the overshoot (white edges/white spots) and undershoot (black edges/black spots) of the sharpened image, and suppress noise enhancement. ### API Reference - ss_mpi_isp_set_sharpen_attr: Set image sharpening attributes. - ss_mpi_isp_get_sharpen_attr: Get image sharpening attributes. #### ss_mpi_isp_set_sharpen_attr 【Description】 Set image sharpening attributes. 【Syntax】 td_s32 ss_mpi_isp_set_sharpen_attr(ot_vi_pipe vi_pipe, const ot_isp_sharpen_attr *shp_attr); 【Parameters】

Images typically need to be displayed on display devices. CRT monitors generally have a dynamic range of less than 50dB, while the dynamic range of WDR-synthesized data can reach 120dB. If displayed directly on a CRT monitor, dynamic range mismatch would cause loss of detail in areas with very high or very low brightness. The purpose of the DRC algorithm is to ensure that both the observer of the real scene and the observer of the display device have the same visual experience. The DRC algorithm compresses high dynamic range images into the dynamic range of the display device while preserving as much of the original image's details and contrast as possible. Figure 1 Dynamic Range Compression ### API Reference - ss_mpi_isp_set_drc_attr: Set dynamic range compression parameters. - ss_mpi_isp_get_drc_attr: Get dynamic range compression parameters. #### ss_mpi_isp_set_drc_attr 【Description】 Set dynamic range compression parameters. 【Syntax】 td_s32 ss_mpi_isp_set_drc_attr(ot_vi_pipe vi_pipe, const ot_isp_drc_attr *drc_attr); 【Related Topics】 ss_mpi_isp_get_drc_attr ## Mesh Shading ### Function Description The LSC module is mainly used to handle vignetting (shadows around the lens edges) caused by non-uniform optical refraction of the lens. Currently popular processing methods include the Radial (coaxial circle) method and the Mesh (grid) method. In Hi3403V100, the Mesh Shading algorithm uses the Mesh (grid) method for lens calibration/correction. The algorithm divides the entire Bayer domain image into 32x32 sub-regions, each approximately equal in size. During data processing, the algorithm processes each channel in the RAW domain separately. ### API Reference - ss_mpi_isp_set_mesh_shading_attr: Set Mesh Shading algorithm parameters. - ss_mpi_isp_get_mesh_shading_attr: Get Mesh Shading algorithm parameters. - ss_mpi_isp_set_mesh_shading_gain_lut_attr: Set Mesh Shading gain LUT attributes. - ss_mpi_isp_get_mesh_shading_gain_lut_attr: Get Mesh Shading gain LUT attributes. ## Defect Pixel ### Function Description The Defect Pixel Correction module uses dynamic and static methods to detect and correct defective pixels in images. Due to sensor process or material issues, some pixel points may have abnormal brightness (bright spots or dark spots). The Defect Pixel module has two working modes: Static Defect Pixel Correction (SDPC) and Dynamic Defect Pixel Correction (DDPC). Static correction is applied to fixed defective pixel points by calibrating coordinates using a calibration tool, storing them in a static defect pixel table, and using this table for correction. Dynamic correction uses neighborhood pixel comparison algorithms to dynamically detect and correct possible defective pixel points in real time. ### API Reference - ss_mpi_isp_set_dpc_attr: Set defect pixel correction attributes. - ss_mpi_isp_get_dpc_attr: Get defect pixel correction attributes. - ss_mpi_isp_set_dpc_dynamic_attr: Set DPC dynamic attributes. - ss_mpi_isp_get_dpc_dynamic_attr: Get DPC dynamic attributes. ## Crosstalk Removal ### Function Description The purpose of cross-color removal is to reduce the color cross-talk phenomenon in images caused by the increasing photosensitivity of pixel sensors, which leads to increased optical crosstalk between different color channels of adjacent pixels. Eliminating or reducing this crosstalk improves image color reproduction. ### API Reference - ss_mpi_isp_set_xtalk_attr: Set cross-color removal attributes. - ss_mpi_isp_get_xtalk_attr: Get cross-color removal attributes. ## Denoising Algorithm ### Function Description The ISP denoising algorithm can be divided into two processing stages. In the RAW domain denoising stage, since valuable signal and noise are mixed together, reducing noise inevitably causes some loss of detail. To balance this, current denoising algorithms apply different denoising intensity levels based on the image brightness. The algorithm independently adjusts the denoising strength for both the luminance channel and the combined chrominance channel, making denoising more flexible. In the YUV domain denoising stage, the image has been demosaiced and separated into luminance and chrominance information. Human vision is more sensitive to luminance noise than chrominance noise, so the denoising strategy differs: luminance noise is mainly removed through 2D + 3D filtering, while chrominance noise is primarily handled by median filtering. ### API Reference - ss_mpi_isp_set_nr_attr: Set denoising attributes. - ss_mpi_isp_get_nr_attr: Get denoising attributes. - ss_mpi_isp_set_nr_attr_ex: Set denoising extended attributes. - ss_mpi_isp_get_nr_attr_ex: Get denoising extended attributes. - ss_mpi_isp_set_ynr_attr: Set luminance denoising attributes. - ss_mpi_isp_get_ynr_attr: Get luminance denoising attributes. - ss_mpi_isp_set_cnr_attr: Set chrominance denoising attributes. - ss_mpi_isp_get_cnr_attr: Get chrominance denoising attributes. ## Dehaze ### Function Description Dehaze is implemented by dynamically changing the contrast and brightness of the image. The image is divided into blocks, the pixel values within each block are counted, the fog concentration is estimated, and the dehazing intensity is adjusted according to a local adaptive curve. Dehaze has manual and automatic modes, and the dehazing intensity can be adjusted in both modes. ### API Reference - ss_mpi_isp_set_dehaze_attr: Set dehaze attributes. - ss_mpi_isp_get_dehaze_attr: Get dehaze attributes. ## False Color Removal ### Function Description High-frequency components can easily cause high-frequency aliasing during image interpolation. When a lens is pointed at a resolution test chart and there is no OLPF on the sensor surface, false colors are prone to appear in the high-frequency part of the resolution. False color removal mainly refers to removing the false color phenomenon caused by interpolation errors in the high-frequency part. Increasing the false color removal intensity will reduce the false color phenomenon but may cause normal colors to become grayscale. ### API Reference - ss_mpi_isp_set_false_color_attr: Set false color removal attributes. - ss_mpi_isp_get_false_color_attr: Get false color removal attributes. ## Demosaic ### Function Description The Demosaic module converts input Bayer data into RGB data. In the Bayer data obtained from the CFA (Color Filter Array), each pixel can only obtain one color component among the three primary colors R, G, and B. To obtain a color image, it is necessary to use the color component values of the current pixel and surrounding pixels to estimate the other two missing component values of the current point. This module can output higher image resolution, significantly improving weak details and sharpness. Figure 1 Demosaic Principle Diagram ### API Reference - ss_mpi_isp_set_demosaic_attr: Set Demosaic attributes. - ss_mpi_isp_get_demosaic_attr: Get Demosaic attributes. ## BayerSharpen ### Function Description The BayerSharpen module is used to enhance image clarity, including adjusting the sharpening attributes of image edges and enhancing image details and textures. It can also control the sharpening intensity of image edges in the horizontal and vertical directions. Additionally, it can control the overshoot and undershoot of the sharpened image and suppress noise enhancement. ### API Reference - ss_mpi_isp_set_bayer_sharpen_attr: Set BayerSh sharpen attributes. - ss_mpi_isp_get_bayer_sharpen_attr: Get BayerSharpen attributes. ## Black Level ### Function Description Black level usually refers to the brightness value that the sensor still outputs when there is no external light input. ISP needs to subtract this brightness value for color processing. ### API Reference - ss_mpi_isp_set_black_level_attr: Set black level attributes. - ss_mpi_isp_get_black_level_attr: Get black level attributes. ## FPN Removal ### Function Description The sensor converts optical signals into electrical signals and outputs the image through millions of ADC devices. Variations in the photodiode size, doping concentration, contamination during production, and MOS field-effect transistor parameter deviations in each pixel structure can cause changes in the pixel output signal. The noise caused by these deviations is fixed for a given individual pixel, known as Fixed Pattern Noise (FPN). The FPN removal module is used to eliminate these fixed pattern noises. ### API Reference - ss_mpi_isp_set_fpn_attr: Set FPN removal attributes. - ss_mpi_isp_get_fpn_attr: Get FPN removal attributes. ## CA ### Function Description The color adjustment module supports color gamut adjustment in YUV space. This module has two modes: CA mode and CP mode (thermal imaging coloring). Only one of the two can be used at a time. In CA mode, a pixel point (Y, U, V) can be mapped to another pixel point (Y', U', V') through the following formula. ### API Reference - ss_mpi_isp_set_ca_attr: Set CA attributes. - ss_mpi_isp_get_ca_attr: Get CA attributes. ## CAC ### Function Description Chromatic Aberration refers to the optical phenomenon where a lens cannot focus light of all wavelengths to the same point. It is a lens-related defect mainly caused by different wavelengths of light having different refractive indices (dispersion). Chromatic aberration can be divided into two types: - Axial Chromatic Aberration: Lens has different refractive indices for different light, so different colors focus on different focal planes. Large aperture lenses are prone to this type of aberration; stopping down the aperture can reduce axial chromatic aberration. - Lateral Chromatic Aberration: Different wavelengths of light are magnified differently by the lens. ### API Reference - ss_mpi_isp_set_cac_attr: Set CAC attributes. - ss_mpi_isp_get_cac_attr: Get CAC attributes. ## WDR ### Function Description Since traditional image sensors have a relatively low dynamic range, capturing high dynamic range scenes with traditional sensors results in either overexposed bright areas (losing many details) or underexposed dark areas (details difficult to discern). To record every detail of a high dynamic range scene, multi-exposure image synthesis can be used. The purpose of the WDR algorithm is to synthesize the long and short frame data output from the sensor into one frame of 20-bit data. Long and short frame data is first buffered in DDR and then simultaneously sent to the WDR module for synthesis. The synthesis principle is optimal selection: for brighter areas, short frame data is used; for darker areas, long frame data is used. Figure 1 WDR Synthesis Diagram ### API Reference - ss_mpi_isp_set_wdr_attr: Set WDR attributes. - ss_mpi_isp_get_wdr_attr: Get WDR attributes. - ss_mpi_isp_set_wdr_compensate_attr: Set WDR compensation attributes. - ss_mpi_isp_get_wdr_compensate_attr: Get WDR compensation attributes. ## AWB Mass Production Calibration Tool ### Function Description > CAUTION: > AWB mass production calibration results correct individual differences by modifying the static white balance coefficients of AWB. Therefore, users modifying static white balance coefficients through the MPI interface will invalidate the mass production calibration. The AWB mass production calibration tool implements correction of AWB difference parameters for different devices during mass production. Users can call the ss_mpi_isp_get_lightbox_gain interface to obtain the average Rgain and Bgain values of several blocks in the center of the image under a uniform light source in the color temperature range of 4500K to 6500K with a uniform background environment. These AWB gains are used to correct AWB difference parameters for different devices during mass production. ## ColorTone ### Function Description The ColorTone module is used to adjust the overall color tone of the image. ### API Reference -
