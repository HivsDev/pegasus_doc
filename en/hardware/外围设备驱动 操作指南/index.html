    GMAC Operation Guide ### Operation Examples > Note: >By default, all related GMAC modules are compiled into the kernel, so no loading operation is required. Please skip directly to the step for configuring the IP address. Under the kernel, using the network interface involves the following aspects: - The GMAC module supports TSO and it is enabled by default. If the user wishes to disable TSO, it can be turned off using the ethtool tool. The methods for toggling TSO are as follows: - Disable TSO: ./ethtool -K eth0 tx off - Enable TSO: ./ethtool -K eth0 tx on TSO (TCP Segment Offload) overview: - TSO (TCP Segmentation Offload) is a technology that uses the network card to segment large data packets, reducing CPU load. It is also known as LSO (Large Segment Offload). If the data packet type is only TCP, it is called TSO. If the hardware supports TSO, it must also support hardware TCP checksum calculation and Scatter Gather functionality. TSO is implemented through a combination of software and hardware. Specifically, the hardware can fragment large data packets and attach relevant headers to each fragment. - When the chip uses TSO, part of the processing work originally handled by the CPU is transferred to the network card, reducing CPU pressure and improving performance. - Configure IP address and subnet mask ifconfig eth0 xxx.xxx.xxx.xxx netmask xxx.xxx.xxx.xxx up - Set default gateway route add default gw xxx.xxx.xxx.xxx - Mount NFS mount -t nfs -o nolock xxx.xxx.xxx.xxx:/your/path /mount-dir - Upload/download files via tftp in the shell Prerequisite is that the tftp service software is running on the server side. - Download file: tftp -r XX.file serverip -g Where: XX.file is the file to download, serverip is the IP address of the server where the file resides. - Upload file: tftp -l xx.file remoteip -p Where: xx.file is the file to upload, remoteip is the IP address of the server to which the file is uploaded. ### IPv6 Description IPv6 is disabled by default in the release package. Once the timer reaches the flow control time, transmission resumes. Alternatively, if a flow control frame with a flow control time of 0 is received from the peer during the wait, transmission also resumes. #### Flow Control Configuration<a name="ZH-CN_TOPIC_0000002512103583"></a> The flow control configuration method is as follows: cd open_source/linux/linux-4.19.y cp arch/arm64/configs/Hi3403V100_defconfig .config make ARCH=arm64 CROSS_COMPILE=aarch64-mix210-linux- menuconfig The corresponding options are: Device Drivers ---> [] Network device support ---> [] Ethernet driver support ---> [] Vendor devices <> eth gmac family network device support ---> User-configurable flow control parameters: - CONFIG\_RX\_FLOW\_CTRL\_SUPPORT: Enables or disables reception of flow control frames. - CONFIG\_TX\_FLOW\_CTRL\_SUPPORT: Enables or disables transmission of flow control frames. - CONFIG\_TX\_FLOW\_CTRL\_ACTIVE\_THRESHOLD: The activation watermark for sending flow control frames. When the number of available descriptors in the receive queue falls below this value, the logic to send flow control frames is initiated. - CONFIG\_TX\_FLOW\_CTRL\_DEACTIVE\_THRESHOLD: The deactivation watermark for sending flow control frames. When the number of available descriptors in the receive queue is greater than or equal to this value and the system is currently in a flow control state, the current flow control state is released. > **Note:** >The activation watermark value must be less than the deactivation watermark. #### ethtool Configuration Interface<a name="ZH-CN_TOPIC_0000002480063656"></a> Users can enable or disable the flow control function through the standard ethtool tool interface. Command `ethtool -a eth0` to view the flow control status of the eth0 interface; output:

    ./ethtool -a eth0¶¶

    Pause parameters for eth0: Autonegotiate: on RX: on TX: on Here, RX flow control is enabled, and TX flow control is enabled. Users can enable or disable TX flow control with the following commands:

    ./ethtool -A eth0 tx off (Disable TX flow control)¶¶

    ./ethtool -A eth0 tx on (Enable TX flow control)¶¶

    ### Ipfrag Parameter Configuration<a name="ZH-CN_TOPIC_0000002512063515"></a> When receiving UDP data streams with high bandwidth and large packet sizes under unstable network conditions, fragment memory may be exhausted, causing the protocol stack to proactively drop packets. It is recommended to appropriately increase the values of the IP parameters `ipfrag_high_thresh` and `ipfrag_low_thresh`. The current default values are 4194304 and 3145728. Modification method: xxx is the value to set ~ # echo xxx > /proc/sys/net/ipv4/ipfrag_high_thresh ~ # echo xxx > /proc/sys/net/ipv4/ipfrag_low_thresh Parameter descriptions: `ipfrag_high_thresh` sets the maximum amount of memory in the kernel that can be used for IP fragment reassembly. When this maximum boundary is reached, the handler responsible for fragment reassembly will discard all pending IP fragments until the memory usage falls back to the minimum boundary value `ipfrag_low_thresh`. ### Dual Ethernet Port Configuration<a name="ZH-CN_TOPIC_0000002512063533"></a> When the application scenario requires dual Ethernet ports, the default SDK and demo board do not support dual ports.
