Images typically need to be displayed on display devices. CRT monitors generally have a dynamic range of less than 50dB, while the dynamic range of WDR-synthesized data can reach 120dB. If displayed directly on a CRT monitor, dynamic range mismatch would cause loss of detail in areas with very high or very low brightness. The purpose of the DRC algorithm is to ensure that both the observer of the real scene and the observer of the display device have the same visual experience. The DRC algorithm compresses high dynamic range images into the dynamic range of the display device while preserving as much of the original image's details and contrast as possible. **Figure 1** Dynamic Range Compression [](figures/Dynamic Range Compression) ### API Reference - [ss\_mpi\_isp\_set\_drc\_attr](#ZH-CN_TOPIC_0000002470925106): Set dynamic range compression parameters.

In Hi3403V100, the Mesh Shading algorithm uses the Mesh (grid) method for lens calibration/correction. The algorithm divides the entire Bayer domain image into 32x32 sub-regions, each approximately equal in size. Due to sensor process or material issues, some pixel points may have abnormal brightness (bright spots or dark spots). The Defect Pixel module has two working modes: Static Defect Pixel Correction (SDPC) and Dynamic Defect Pixel Correction (DDPC). Static correction is applied to fixed defective pixel points by calibrating coordinates using a calibration tool, storing them in a static defect pixel table, and using this table for correction. Dynamic correction uses neighborhood pixel comparison algorithms to dynamically detect and correct possible defective pixel points in real time. ### API Reference - [ss\_mpi\_isp\_set\_dpc\_attr](#ZH-CN_TOPIC_0000002471084902): Set defect pixel correction attributes.

Eliminating or reducing this crosstalk improves image color reproduction. ### API Reference - [ss\_mpi\_isp\_set\_xtalk\_attr](#ZH-CN_TOPIC_0000002470925054): Set cross-color removal attributes.
- [ss\_mpi\_isp\_get\_xtalk\_attr](#ZH-CN_TOPIC_0000002504085025): Get cross-color removal attributes. ## Denoising Algorithm ### Function Description The ISP denoising algorithm can be divided into two processing stages. In the RAW domain denoising stage, since valuable signal and noise are mixed together, reducing noise inevitably causes some loss of detail. To balance this, current denoising algorithms apply different denoising intensity levels based on the image brightness. The algorithm independently adjusts the denoising strength for both the luminance channel and the combined chrominance channel, making denoising more flexible. In the YUV domain denoising stage, the image has been demosaiced and separated into luminance and chrominance information. Human vision is more sensitive to luminance noise than chrominance noise, so the denoising strategy differs: luminance noise is mainly removed through 2D + 3D filtering, while chrominance noise is primarily handled by median filtering. ### API Reference - [ss\_mpi\_isp\_set\_nr\_attr](#ZH-CN_TOPIC_0000002504085001): Set denoising attributes.

When a lens is pointed at a resolution test chart and there is no OLPF on the sensor surface, false colors are prone to appear in the high-frequency part of the resolution. False color removal mainly refers to removing the false color phenomenon caused by interpolation errors in the high-frequency part. In the Bayer data obtained from the CFA (Color Filter Array), each pixel can only obtain one color component among the three primary colors R, G, and B. To obtain a color image, it is necessary to use the color component values of the current pixel and surrounding pixels to estimate the other two missing component values of the current point. This module can output higher image resolution, significantly improving weak details and sharpness. **Figure 1** Demosaic Principle Diagram [](figures/Demosaic Principle Diagram) ### API Reference - [ss\_mpi\_isp\_set\_demosaic\_attr](#ZH-CN_TOPIC_0000002504084729): Set Demosaic attributes.

It is a lens-related defect mainly caused by different wavelengths of light having different refractive indices (dispersion). Chromatic aberration can be divided into two types: - Axial Chromatic Aberration: Lens has different refractive indices for different light, so different colors focus on different focal planes. Large aperture lenses are prone to this type of aberration; stopping down the aperture can reduce axial chromatic aberration.
- Lateral Chromatic Aberration: Different wavelengths of light are magnified differently by the lens. ### API Reference - [ss\_mpi\_isp\_set\_cac\_attr](#ZH-CN_TOPIC_0000002504084711): Set CAC attributes.
- [ss\_mpi\_isp\_get\_cac\_attr](#ZH-CN_TOPIC_0000002470925176): Get CAC attributes. ## WDR ### Function Description Since traditional image sensors have a relatively low dynamic range, capturing high dynamic range scenes with traditional sensors results in either overexposed bright areas (losing many details) or underexposed dark areas (details difficult to discern). To record every detail of a high dynamic range scene, multi-exposure image synthesis can be used. The purpose of the WDR algorithm is to synthesize the long and short frame data output from the sensor into one frame of 20-bit data. Long and short frame data is first buffered in DDR and then simultaneously sent to the WDR module for synthesis. The synthesis principle is optimal selection: for brighter areas, short frame data is used; for darker areas, long frame data is used. **Figure 1** WDR Synthesis Diagram [](figures/WDR Synthesis Diagram) ### API Reference - [ss\_mpi\_isp\_set\_wdr\_attr](#ZH-CN_TOPIC_0000002504084973): Set WDR attributes.
