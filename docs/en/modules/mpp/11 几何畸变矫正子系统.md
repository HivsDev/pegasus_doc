--- # Geometric Distortion Correction Subsystem

## Overview The Geometric Distortion Correction (GDC) subsystem implements the following functions: fisheye correction (including 360-degree panorama, 180-degree panorama, and Normal correction modes), LDC (Lens Distortion Correction) correction, spread, rotation, PMF correction, output image stitching, and output image background color filling. > **Note:**[¶](#overview-the-geometric-distortion-correction-gdc-subsystem-implements-the-following-functions-fisheye-correction-including-360-degree-panorama-180-degree-panorama-and-normal-correction-modes-ldc-lens-distortion-correction-correction-spread-rotation-pmf-correction-output-image-stitching-and-output-image-background-color-filling-note "锚链接")

> - Only Hi3403V100 support this module. ## Functional Description ### Basic Concepts - **job** A structure for GDC to manage tasks. A job can contain multiple tasks. GDC ensures that tasks are executed in the order they are added to the job and that all tasks in a job are submitted to hardware for execution at once. Users can set the maximum number of jobs supported by GDC through the GDC module parameter (g\_max\_gdc\_job). The number of jobs ranges from [20, 400], with a default of 128. - **task** Performs one or more specific operations on an image, such as fisheye correction or LDC correction. Users can set the maximum number of tasks supported by GDC through the GDC module parameter (g\_max\_gdc\_task). The number of tasks ranges from [20, 800], with a default of 200. - **node** The minimum execution unit of GDC hardware (from a hardware perspective). One task corresponds to one node. A node contains the information required by the hardware, such as the source address, destination address, operation type, etc., organized in the format required by the hardware. Users can set the maximum number of nodes supported by GDC through the GDC module parameter (g\_max\_gdc\_node). The number of nodes ranges from [20, 800], with a default of 200. ### Functional Description The main functions of GDC include fisheye correction (including output image stitching after correction, output image background color filling, etc.), lens distortion correction, spread, rotation, PMF correction, etc. - **Fisheye Correction** GDC supports fisheye correction processing on an image. > **Note:** > - Fisheye correction supports three correction modes: 360-degree panorama mode, 180-degree panorama mode, and normal mode. > - 360-degree panorama mode supports floor-mount and ceiling-mount; 180-degree panorama mode supports wall-mount; normal mode supports floor-mount, ceiling-mount, and wall-mount. > - During fisheye algorithm processing, the algorithm computes block sizes based on the output image width/height, out\_radius, and mode. The more blocks, the greater the time consumption. The maximum out\_radius configuration is half of max(output image width, output image height). If this is exceeded, the number of blocks will increase significantly, and hardware time consumption will also increase significantly. GDC supports scaling while performing fisheye correction on an image. Fisheye scaling results are not optimal. It is recommended to use typical parameter settings and try not to use GDC for scaling fisheye-corrected images. > **Note:** > Fisheye input and output image specifications are shown in [Table 1](#_Ref48226574). During fisheye correction, images can be scaled within the allowable range of the output image. - **Lens Distortion Correction** GDC supports lens distortion correction processing on an image. - **Spread** GDC supports far-end spread processing on an image. > **Note:** > When using GDC for lens distortion correction or spread processing, the input and output image sizes must be the same. Simultaneous scaling is not supported. - **Rotation** GDC supports fixed-angle or arbitrary-angle rotation of a frame. Fixed-angle rotation does not support compressed output. Fixed-angle rotation is only performed by GDC when combined with LDC, DIS, or spread tasks; otherwise, it is performed by VGS. - **PMF Correction** GDC supports PMF correction processing on a frame, enabling projective transformation of planar images. For details, refer to the "GDC Debugging Guide." - **Compression** GDC only supports LINEAR format non-compressed image input and only supports LINEAR format compressed/non-compressed image output. - **Output Image Stitching** GDC supports performing FISHEYE correction on multiple regions of an image separately, and then stitching the corrected images for output. For non-compressed output, the x coordinate of each region requires 16-pixel alignment, and the y coordinate requires 2-pixel alignment. For compressed output, the x coordinate of each region requires 256-pixel alignment, and the width of each region requires 256-pixel alignment, or must equal the difference between the output image width and the x coordinate of that region. The y coordinate always requires 2-pixel alignment. > **Note:** > Output image stitching here refers to correcting up to 4 regions separately within the same VI or VPSS channel, and stitching the corrected images into one output image. Fisheye correction processing outputs from different channels do not support stitching output. - **Output Image Background Color** During FISHEYE correction, the output image can be filled with a background color for output with seams in stitching. For seamless stitching output, it is recommended to disable the background color to save performance. **Table 1** Hi3403V100 GDC Hardware Specifications
