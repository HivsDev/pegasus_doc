--- # System overview ## Overview The Media Process Platform (MPP) is a software platform that lets
applications develop quickly. It hides chip-specific low-level
processing from the application and exposes the **MPP Programe
Interface** (MPI) directly. MPP supports rapid development of features such as: video input
capture, H.264 / MJPEG / JPEG encoding, H.264 / H.265 decoding, video
display, video pre-processing (denoising, enhancement, sharpening,
de-interlacing), image splicing, geometric correction, OSD overlay on
the encoded bitstream, video detection, intelligent analysis, audio
capture and playback, and audio codec. ## System architecture The typical MPP-based system is layered as follows: - **Hardware layer.** The chip plus the necessary periphery: Flash, DDR (Double Data-Rate), video sensor or AD, audio AD, etc. - **Operating-system layer.** Based on Linux: | Linux version | Solution | | --- | --- | | Linux 4.19.y | / / / / Hi3403V100 | | Linux 5.10.y | | - **Media-processing platform.** Built on the OS layer, drives the chip to perform the media-processing functions, hides hardware details from applications, and exposes AP Is for application use. - **Application layer.** Calls MPP AP Is to implement specific features. MPP is split into a set of subsystems: | Subsystem | Responsibility |
|---|---|
| `sys_init` | Global init, MMZ memory pool, system control |
| **VI** (Video Input) | Pull frames from sensor / MIPI-CSI |
| **ISP** (Image Signal Processor) | RAW → YUV conversion, AWB, AE, gamma, denoise |
| **VPSS** (Video Process Sub System) | Scaling, format conversion, region cropping |
| **VENC** (Video Encoder) | H.264 / H.265 / JPEG encoding |
| **VDEC** (Video Decoder) | H.264 / H.265 / JPEG decoding |
| **VO** (Video Output) | HDMI / MIPI-DSI / VGA / BT.1120 output |
| **AI** (Audio Input), **AO** (Audio Output) | Audio capture / playback |
| **AENC** (Audio Encoder), **ADEC** (Audio Decoder) | Audio codec |
| **GFBG** (Graphics Frame Buffer Group) | Graphics framebuffer management |
| **TDE** (Two-Dimensional Engine) | 2D graphics acceleration |
| **RGN** (Region) | Overlay / shielding regions |
| **HNR** (Heterogeneous NR) | High-quality denoising |
| **DIS** (Digital Image Stabilization) | Electronic image stabilization |
| **GDC** (Geometric Distortion Correction) | Fish-eye / geometric correction |
| **SVP** (Smart Vision Platform) | NPU-based AI inference |
| **IVE** / **IVS** | Classic-CV / video-structuring acceleration | ## Pipeline binding MPP modules talk to each other via a **bind** mechanism: e.g. bind VI
output → VPSS input → VENC input. Once bound, frames flow without
application intervention. Use `ss_mpi_sys_bind` / `ss_mpi_sys_un_bind`
to wire / unwire pipelines at runtime. ## Memory model MPP uses **MMZ** (Media Memory Zone) — a physically contiguous reserved
DDR region that frames and bitstreams flow through, avoiding the
fragmentation problems of normal Linux page allocation. For an introduction to MMZ allocation in user / kernel code, see
[Memory layout](../../../soc-linux/memory-layout/).
