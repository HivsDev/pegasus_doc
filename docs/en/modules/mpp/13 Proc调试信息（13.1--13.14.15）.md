--- # proc Debug Information ## Overview
Debug information is implemented via the Linux proc file system, which reflects the current system runtime status in real time. The recorded information can be used for problem diagnosis and analysis. **File Directory** /proc/umap **File List**

| File Name | Description |
| --- | --- |
| sys | Records the current usage of the SYS module. |
| logmpp | Records the current debug level of each module, for internal debugging use. |
| chnl | CHNL module status. |
| vgs | Video scaling processing unit status information. |
| h265e | During H.265 encoding, statistics of encoding attributes, status, and historical information for each channel. |
| h264e | During H.264 encoding, statistics of encoding attributes, status, and historical information for each channel. |
| jpege | During JPEG encoding, statistics of encoding attributes, status, and historical information for each channel. |
| rgn | Region management information for video overlay OSD. |
| venc | Video encoder information. |
| vdec | Video decoder information. |
| vfmw\_ctrl | Common information for each channel during video decoding. |
| vfmw\_stm | Stream information for each channel during video decoding. |
| vfmw\_syn | Syntax information for each channel during video decoding. |
| vfmw\_dec | Hardware configuration information for each channel during video decoding. |
| vpss | Video preprocessing module information. |
| aenc | Audio encoding information. |
| adec | Audio decoding information. |
| acodec | Acodec volume information. |
| vda | Video detection and analysis module information. |
| avs | Stitching processing module information |

**Information Viewing Methods** - On the console, the cat command can be used to view information, e.g., cat /proc/umap/venc; other common file operation commands can also be used, e.g., cp /proc/umap/ ./ -rf, to copy the proc files under umap to the current directory.
- In applications, the above files can be treated as ordinary read-only files for read operations, e.g., using fopen, fread, etc. > **Note:** >The following 2 cases should be noted for parameter descriptions:

> - For parameters with values {0, 1}, if the specific value-meaning correspondence is not listed, then 1 indicates affirmative and 0 indicates negative.
> - For parameters with values {aaa, bbb, ccc}, if the specific value-meaning correspondence is not listed, the parameter meaning can be directly determined from the values aaa, bbb, or ccc. ## SYS

- cat /proc/umap/logmpp is used to obtain kernel-mode logmpp level information.
- cat /proc/umap/logmpp\_user is used to obtain user-mode logmpp level information.
- Modifying logmpp level: - Use the echo command to modify the debug level of a module, e.g.: echo "venc=4" > /proc/umap/logmpp (logmpp\_user for user mode); - Use: echo "all=4" > /proc/umap/logmpp (logmpp\_user for user mode) to modify the debug level of all modules. - View logmpp: open using cat /dev/logmpp, exit by pressing Ctrl+c. **Parameter Description**
