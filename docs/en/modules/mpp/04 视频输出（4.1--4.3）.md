> - Video Writeback The writeback device is called WD (Write Back Channel Device). The writeback function captures video layer and device-level video data, which can be used for display and encoding. The use of the writeback device depends on the device being written back from. It can write back device-level (after video layer and graphic layer overlay) video data, or it can write back video layer data. - Channel - The SDK manages channels under the video layer. Multiple videos can be displayed on one video layer. Each video display area is called a channel. Video is restricted within a channel, and channels are restricted within a video layer. For a given video layer, all channels on it are independent. Similarly, channels on different video layers are also independent. Channel numbering does not have continuity across layers. - For system binding, the video layer number and channel number should be used for binding configuration. Additionally, when a writeback image is used as an output source, the device number of the writeback content and channel 0 should be used for binding configuration. - In SINGLE mode: The source image is scaled to the channel area size. - In MULTI mode: Displays the intersection of the source image and the channel display area. - Video Layer Scaling - When the video layer supports scaling: if img\_size < display\_rect, img\_size is scaled up (maximum 15x) to display\_rect for display; if img\_size > display\_rect, the display shows the intersection of img\_size and display\_rect. - When the video layer does not support scaling: the display shows the intersection of img\_size and display\_rect.
> - For Hi3403V100, all video layers do not support scaling in MULTI mode. For , video layers that support scaling only support full-screen scaling in MULTI mode, i.e., display\_rect equals the device size; otherwise, the display shows the intersection of img\_size and display\_rect. For , the video layer scaling function is not supported. >[](../../../../multimedia/mpp/public_sys-resources/icon-notice.gif) **Notice:** >- Clipping refers to when the width and height of the canvas size img\_size in the video layer attributes are greater than the width and height of the display device resolution display\_rect, the canvas is clipped to match the display resolution.
> - Scaling refers to when the width and height of the display device resolution display\_rect in the video layer attributes are greater than the width and height of the canvas size img\_size, the canvas is scaled up to match the display resolution. - Channel Priority VO software supports multiple channels outputting and displaying simultaneously. Output images are overlaid according to priority order. When channel images overlap, the higher priority image is displayed on top. If all channels have the same priority, the one with the larger channel number has a higher default priority. - Resolution Resolution mainly involves the following 3 concepts: - Device resolution refers to the effective pixel count of the device output, determined by the device timing. - Display resolution refers to the effective display area of the image on the display device, determined by the display\_rect member in the video layer attributes. - Canvas resolution refers to the memory area of output video data used during VGS compositing, determined by the img\_size in the video layer attributes. This definition does not apply in pass-through mode. - Image resolution refers to the effective pixel count of the image itself, determined by the actual displayed image resolution. - Partial Zoom - The channel supports zooming in on a portion of the image for display. The source area for zoomed display is cropped from the source image, and the target area for zoomed display is the channel size of the display channel. - The maximum zoom ratio supported by partial zoom in both horizontal and vertical directions is 16x. - Video Layer or Graphic Layer Binding - Video layer or graphic layer binding refers to binding a specific video layer or graphic layer to a device. For dynamically bindable video or graphic layers, see [ss\_mpi\_vo\_bind\_layer](#ZH-CN_TOPIC_0000002470925692) for the default binding device. - Single-frame Passthrough Mode Single-frame passthrough mode means that in SINGLE mode, the image in the VO channel is displayed directly without going through the VGS module, saving one VGS transfer process, and VO does not need to allocate private memory. However, the following conditions must be met for passthrough mode to be selected: - VO channel is enabled, and only one channel is displayed; - VO does not display OSD and COVER; - VO does not have line drawing enabled; - VO does not have corner frame enabled; - VO does not have border enabled (or border is enabled with top, bottom, left, and right widths all set to 0); - VO does not have anti-flicker enabled; - VO does not have electronic zoom; - VO does not have mirror enabled; - VO channel image pixel format is consistent with the video layer pixel format; - VO channel image video format is LINEAR (including LINEAR DISCRETE format); - VO channel image size (Frame Size) = channel size (Chn Rect) = canvas size (Image Size); - The starting address of the video layer input image frame must be 16-byte aligned; - When the video layer input image compression format meets the VO video layer hardware decompression capability, see [Table 2](#table123621823184220) for the specific video layer supported decompression capability; - The data received by VO does not have the following characteristics: compact segment compression with a mirror frame flag. The mirror frame flag can be set through the direct upstream module of VO (such as the VPSS interface ss\_mpi\_vpss\_set\_chn\_attr). (Only applicable to Hi3403V100) If passthrough mode conditions are not met, since VO does not allocate memory by default, the [ss\_mpi\_vo\_set\_video\_layer\_attr](#ZH-CN_TOPIC_0000002471085684) interface must be called to set the buffer length for memory allocation before enabling the video layer. Otherwise, VO will not output video display. - Decompression and Compression - Video data decompression. When the input source image is compressed data (in line compression format), VO performs decompression for display. The decompression function is a video layer feature. In MULTI mode (see [Basic Concepts](#ZH-CN_TOPIC_0000002471085714) "SINGLE Mode and MULTI Mode"), the compression modes or compression mode combinations supported by all video layers are shown in [Table 3](#_Ref34158714). - Low Power Strategy The VO module uses a low power strategy. After the module is loaded or the device is disabled, the VO clock is turned off. Manually reading or writing VO module registers at this point may result in read/write errors or system hangs. - Rotation VO supports rotation operations on images before they enter the channel. The rotation angle is set on the channel, and rotation applies to the image before it enters the channel. A typical application of rotation is to write back the image from DHD0 and send it to DHD1 for display. If DHD1 is connected to a portrait screen, the channel on DHD1 can be set to rotate 90 or 270 degrees for display. - Input and Output Data Formats VO supports input and output of data in specified formats, where output refers to writing back data to DDR. The supported input and output data formats are shown in the solution's input/output data format table. The PIXEL FORMAT, VIDEO FORMAT, COMPRESS MODE, and DYNAMIC RANGE columns list the supported input/output pixel formats, video formats, compression modes, and video dynamic range respectively. For data formats, refer to Chapter 2 "System Control." - SINGLE Mode and MULTI Mode SINGLE mode and MULTI mode are defined for video layers that support multiple regions, meaning the hardware supports multiple regions. See the "Supported Channel Count - MULTI Mode" column in the solution's video layer specification table. to distinguish from the software channel concept, these are referred to as multiple regions here. SINGLE mode means configuring one display region in hardware, while MULTI mode means configuring multiple display regions (one or more regions) in hardware. Note the distinction between multiple regions and channels. Both SINGLE and MULTI modes can achieve multi-channel display. In SINGLE mode, all channels are organized as one image displayed in one region; in MULTI mode, each channel is independent and exclusively occupies one display region. Additionally, a video layer can only work in one mode. - Pixel Format Conversion - SINGLE mode supports pixel format conversion when not in passthrough mode. If the pixel format of the input image differs from the video layer pixel format, VGS can be called for pixel format conversion. - MULTI mode does not support pixel format conversion. The pixel format of the input image must be the same as the video layer pixel format. - Virtual Device A virtual device is a software concept with no corresponding physical device. - Internal processing flow: Virtual devices only support SINGLE mode, achieve scaling through VGS, and overlay into one complete image. - Sending virtual device images: Virtual devices support obtaining the image of the corresponding video layer through the user-mode thread calling the [ss\_mpi\_vo\_get\_screen\_frame](#ZH-CN_TOPIC_0000002504085581) interface and sending it to other modules. They also support sending virtual device images to other modules in kernel mode via sys\_bind. - Timing constraints: Based on the user-set timing, the virtual device determines the final image size and frame rate after VGS compositing. For example, if the virtual device timing is set to 1080P60, the virtual device will send a 1920x1080@60fps image to other modules. Note that this is a theoretical assumption. The actual output timing of the virtual device also needs to consider VGS performance and the total system workload.
